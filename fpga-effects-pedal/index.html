Overview

— what & why

A digital guitar effects pedal implemented in Verilog, designed to run on a Zynq-7000 SoC FPGA development board (Digilent Arty Z7-20). The goal is a low-latency, modular, reconfigurable pedal that turns a tangle of analog stompboxes into a single programmable effects chain.

Guitar pickups are pre-amplified, biased to 1.65 V, digitized by an I²S ADC Pmod, and pushed into the FPGA fabric. DSP blocks — currently hard clipping and a variable-tap delay line — process the samples sample-accurately. The processed stream is sent back out through an I²S DAC, low-pass filtered to remove clock noise, AC-coupled to strip the DC bias, and amplified to line level. Future iterations move effect chain management and presets onto a Linux control plane running on the ARM cores.

Architecture

— signal path

The processing pipeline is built from composable Verilog modules wired with a ready/valid streaming interface. Each effect block can be inserted or removed without changing its neighbours, which is what makes the chain reconfigurable in principle.

Verilog modules

— what's in sim/

Every block lives in sim/<name>/ with its own testbench. Waveforms (.vcd) are dumped next to the source for inspection in GTKWave.

I²S Input Transceiver i2s_in.v

Receives BCLK / LRCLK / SDATA from the ADC Pmod, deserializes left- and right-channel samples, and emits a 24-bit parallel stream with a ready/valid handshake. Driven by an internal bit-counter so timing is independent of the system clock.

FIFO Buffer fifo_buffer.v

Decouples I²S receive timing from DSP block latency. Small depth (16 entries) is enough to absorb per-stage pipeline stalls without introducing audible delay.

Hard Clip hard_clip.v

Distortion primitive: any sample whose magnitude exceeds a configurable threshold is clamped. Demonstrates the streaming contract: in goes a sample, out comes a clipped sample, ready/valid behavior is preserved.

Delay Line delay.v

Simple single-tap delay used as the foundation for slapback, echo, and (eventually) modulated delay effects. Tap depth and feedback are parameterised at compile time.

Pipeline pipeline.v

End-to-end integration of the blocks above: I²S RX → FIFO → hard clip → FIFO → I²S TX. This is the module that gets synthesised and the one whose testbench is the closest thing to a system-level test today.

AXI-Stream Template axistream_template.v

Reference starting point for new effects that need to plug into the chain. Standardises tdata / tvalid / tready / tlast signals so future blocks (chorus, reverb, compressor) can be added without rewiring neighbours.

Results & status

— what works today
Sample rate
48 kHz
Stereo I²S, 24-bit per sample, target of the end-to-end pipeline.
Effect blocks
2
Hard clip + delay, with a pipeline testbench tying them together.
Toolchain
Icarus
iverilog -g2012 for simulation; VCD dumps viewable in GTKWave.
  • All six modules compile cleanly under iverilog -g2012 and have dedicated testbenches that drive stimulus on a posedge clock with an active-low synchronous reset.
  • End-to-end pipeline simulation runs without X-propagation: I²S RX → FIFO → hard clip → FIFO → I²S TX, with waveform traces captured for review.
  • AXI-Stream template in place so new effect blocks can be added without changing the chain wrapper.
  • Full project context, SWOT analysis, roadmap, BOM, and skills tracker all live in docs/ for design-journal continuity.
  • Next step: port the pipeline to the Arty Z7-20, drive Pmod AD1 / DA3 from real audio, and stand up the Linux-side preset/effect-chain control plane.
