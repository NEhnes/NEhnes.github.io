Low-cost ESP32 audio logger for month-long field deployments in behavioral ecology research.

Problem Statement

— why the system mattered

Field monitoring needed to be low-maintenance.

A Western University biology professor was studying how urban noise affects wildlife behavior on campus. Commercial acoustic monitors solved part of the problem, but their cost and two-week retrieval cycles made long-term deployment cumbersome.

The goal was a practical, affordable logger.

The team delivered a $77.50/unit ESP32-based logger that records month-long field deployments without battery swaps or manual data retrieval, while remaining compatible with ecology analysis tools like Raven Pro and MATLAB.

Technical Approach

— architecture and firmware choices

Boot state machine

The logger initializes I2S and SD, creates a timestamped session folder, then loops through the production duty cycle: record for five minutes, sleep for fifty-five minutes, and repeat.

Initialize I2S receiver with APLL clock and 4 KB × 4 DMA buffers.
Mount MicroSD over conservative SPI settings to reduce electrical noise.
Open WAV files, stream PCM data, and finalize headers on close.
Sleep between sessions while holding required clock and I2S state.

Design constraints

The system had to balance real-time audio capture, file writes, power draw, and late hardware arrival. The firmware architecture intentionally separated I2S capture, WAV writing, SD initialization, and session management so each subsystem could be tested independently.

01

I2S capture

44.1 kHz stereo, 32-bit samples, master receiver mode, APLL enabled.

02

SD storage

VSPI at 1 MHz with organized session folders and timestamped WAV files.

03

Power schedule

5-minute active recording plus 55-minute sleep to target multi-week autonomy.

Design Challenges

— tradeoffs and validation
Challenge 1

Electrical noise in parallel I2S and SPI

High-speed SPI writes created bit errors and clicking artifacts in I2S audio. The team reduced SPI clock speed to 1 MHz, then validated the fix with spectral analysis and a 1 kHz reference tone.

Challenge 2

Firmware base selection under time pressure

A feature-rich reference repo failed to compile, so the project pivoted to a minimal working example. That enabled incremental progress from serial I2S output to WAV generation, SD writes, and session management.

Challenge 3

Signal validation without microphone hardware

A separate ESP32 node generated sine-wave I2S data, letting the firmware validate the WAV encoder and SD pipeline before the real microphone arrived.

Challenge 4

Duty-cycle timing complexity

Compile-time constants allowed the same state machine to run short validation cycles and production 60-minute cycles, reducing regression risk before final submission.

Repository & Build

— setup and configuration

Build locally

The project uses PlatformIO with the Arduino framework, ESP32 board support, and the arduino-audio-tools WAV encoder.

sudo apt install platformio git git clone https://github.com/NEhnes/mic_sd_logger cd mic_sd_logger/src/final_submission pio run -t upload pio device monitor -b 115200

Key configurations

I2S: 44.1 kHz, stereo, 32-bit samples, master RX mode, APLL enabled. SD: 1 MHz SPI clock, GPIO 5 chip select, VSPI pins 18/19/23, FAT32 filesystem. Power: 55-minute sleep with RTC/I2S active and low quiescent current.

Built for real field constraints.

The Wildlife Soundscape Logger shows pragmatic embedded engineering: low cost, clean subsystem boundaries, signal-based validation, and power-aware firmware for unattended ecological monitoring.
